# CI Setup — building Zynbo APKs on every GitHub push

This repo ships a GitHub Actions workflow at `.github/workflows/build-apk.yml` that builds a release-flavoured `.apk` of Zynbo on every push to `main`/`master`, and on every manual run from the Actions tab.

The build is **debug-signed** — fine for personal phones and beta testers, **not** valid for the Play Store. Play Store builds need a release keystore (see [Going to Play Store](#going-to-play-store) below).

---

## 1. One-time GitHub setup (~5 min)

You need to add two secrets to your GitHub repo. The workflow reads them at build time so your real Firebase config never gets committed to git.

### Step 1 — Create the secrets in GitHub
Repo → **Settings → Secrets and variables → Actions → New repository secret**

| Name | Value |
| --- | --- |
| `FIREBASE_OPTIONS_DART` | Paste the **full contents** of your local `lib/firebase_options.dart` (the file generated by `flutterfire configure`) |
| `GOOGLE_SERVICES_JSON` | Paste the **full contents** of your local `android/app/google-services.json` |

> 💡 If you haven't run `flutterfire configure` yet:
> ```bash
> dart pub global activate flutterfire_cli
> cd zynbo && flutterfire configure
> ```
> It will create both files locally. Open each in your editor, copy the entire file, paste as the secret value.

### Step 2 — Add your Android debug SHA-1 to Firebase
This is what lets Google Sign-In work in the built APK.
1. The first GitHub Actions run will build with **its own debug keystore** — different from your local one.
2. After the first successful build, download the APK, open it in `keytool`:
   ```bash
   keytool -printcert -jarfile zynbo-1-abc1234.apk
   ```
   Copy the **SHA1** and **SHA-256** lines.
3. Firebase Console → Project Settings → Your Android app → **Add fingerprint** → paste both → save.
4. Re-download `google-services.json` from Firebase Console, paste new contents into the `GOOGLE_SERVICES_JSON` secret (overwrite).
5. Re-run the workflow.

> 🔧 **Better long-term:** generate your own keystore and use it for every CI build (see [Going to Play Store](#going-to-play-store)). Then the SHA-1 never changes.

## 2. Triggering a build

Three ways:

1. **Push to main** — automatic. Wait ~10–12 min for the first build (deps cached after that).
2. **Manual** — GitHub repo → Actions → "Build Zynbo APK" → **Run workflow** → pick branch → Run.
3. **PR** — does NOT trigger by default (cheap on CI minutes). Add `pull_request` to the workflow if you want this.

---

## 3. Downloading the APK

After a green run:

1. Click into the workflow run.
2. Scroll to **Artifacts** at the bottom.
3. Download `zynbo-apk-<run-number>.zip`.
4. Unzip → you get `zynbo-<run>-<sha>.apk`.
5. Transfer to your Android phone (Drive, email, USB, AirDrop-style apps).
6. Tap to install — first time you'll need to enable **Install unknown apps** for the source app (e.g. Gmail).

The APK works on Android 6+ (minSdk 23).

---

## 4. What the workflow does, in plain English

1. **Checks out** the repo
2. Installs **JDK 17** + **Flutter 3.24.5 stable** (cached)
3. Runs `flutter create .` to fill in missing platform folders **without overwriting** any files we authored
4. Edits `AndroidManifest.xml` to add `INTERNET`, `RECORD_AUDIO`, `READ_MEDIA_IMAGES/AUDIO`, `CAMERA` permissions
5. Writes `firebase_options.dart` + `google-services.json` from secrets
6. Patches `android/build.gradle` + `android/app/build.gradle` for the Google Services plugin + minSdk 23
7. `flutter pub get`
8. `flutter build apk --release`
9. Renames the APK with the run number + commit hash
10. Uploads it as a downloadable artifact (kept for 30 days)

The build is **idempotent** — re-running on the same commit produces the same APK.

## 5. Going to Play Store

To produce a Play-Store-ready signed AAB:

1. Generate a release keystore **once** on your machine:
   ```bash
   keytool -genkey -v -keystore zynbo-release.jks -keyalg RSA -keysize 2048 -validity 10000 -alias zynbo
   ```
2. Add to GitHub Secrets:
   - `ANDROID_KEYSTORE_BASE64` — `base64 zynbo-release.jks` output
   - `ANDROID_KEYSTORE_PASSWORD`
   - `ANDROID_KEY_ALIAS` → `zynbo`
   - `ANDROID_KEY_PASSWORD`
3. Tell me when you're ready and I'll extend the workflow to:
   - Decode the keystore into `android/app/zynbo-release.jks`
   - Write `android/key.properties`
   - Patch `android/app/build.gradle` with a `signingConfigs.release` block
   - Build with `flutter build appbundle --release` instead of `apk`
   - Optionally auto-upload to Google Play via the [Play Store Action](https://github.com/r0adkll/upload-google-play)

That's a separate, deliberate change — don't do it until you've shipped a real beta and are ready to publish.

## 6. Troubleshooting

| Symptom | Likely cause |
| --- | --- |
| `Missing secret FIREBASE_OPTIONS_DART` | Forgot Step 1 above |
| `App crashes on launch — no Firebase App initialized` | `firebase_options.dart` contents stale or the secret has placeholders. Re-run `flutterfire configure` locally and update the secret |
| `Google sign-in fails (ApiException: 10)` | Forgot to add CI build's SHA-1 to Firebase (Step 2) |
| `Gradle build failed: minSdkVersion 21 cannot be smaller than ...` | A library bumped its floor; raise the `sed` line in the workflow that sets minSdk |
| Build is slow (~12 min) | Normal for first run. Subsequent runs use cached Flutter + Gradle and take ~5–6 min |
| APK installs but freezes on splash | Firestore rules not published; check `firestore.rules` is live in Firebase Console |
