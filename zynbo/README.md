# Zynbo тАФ A WhatsApp-style chat app built with Flutter + Firebase

Zynbo is a clean, modern chat application scaffold powered by:

- **Flutter** (Dart) for the cross-platform mobile UI
- **Firebase Authentication** with **Google Sign-In**
- **Cloud Firestore** for user profiles (and future messages)
- **Firebase Storage** (optional, ready for media)

---

## ЁЯОи What's in this scaffold

- **Login screen** тАФ branded, animated Google Sign-In flow
- **Profile setup screen** тАФ collect display name, about, phone (first sign-in only)
- **Home screen** тАФ profile card + chats placeholder, ready for chat list integration
- **`AuthGate`** тАФ automatic routing between Login тЖТ Profile Setup тЖТ Home
- **`AuthService`** тАФ single source of truth for sign-in, sign-out, and Firestore user docs
- **Distinctive design system** тАФ deep teal + lime accents, Space Grotesk type (no purple-gradient AI slop)

## ЁЯЪА Setup (5 minutes)

Since this environment doesn't have the Flutter SDK installed, run these steps **on your local machine** where you have Flutter (`flutter --version` should work):

### 1. Copy this `zynbo/` folder to your dev machine

### 2. Create a real Flutter project shell around it
The scaffold ships only the meaningful files. To generate the platform folders (`android/`, `ios/`, etc.):

```bash
cd zynbo
flutter create . --org com.zynbo --project-name zynbo
```
This will fill in the platform-specific code **without overwriting** the `lib/` and `pubspec.yaml` we authored.

### 3. Install dependencies
```bash
flutter pub get
```

### 4. Wire up Firebase

#### Option A тАФ Recommended: FlutterFire CLI
```bash
dart pub global activate flutterfire_cli
flutterfire configure
```
Select your Firebase project. The CLI will:
- Register the Android/iOS apps
- Drop a real `lib/firebase_options.dart` (overwrites our placeholder)
- Drop `android/app/google-services.json` for you

#### Option B тАФ Manual
1. In **Firebase Console тЖТ Project Settings тЖТ Your apps тЖТ Android**
   - Package name: `com.zynbo.zynbo` (or whatever you used in `flutter create`)
   - Download `google-services.json` тЖТ place at `android/app/google-services.json`
2. Edit `lib/firebase_options.dart` and replace all `REPLACE_ME_*` constants with the values shown in Firebase Console.

### 5. Enable Google Sign-In in Firebase
- Firebase Console тЖТ **Authentication тЖТ Sign-in method тЖТ Google** тЖТ Enable
- Add your **SHA-1** and **SHA-256** debug fingerprints (Android only):
  ```bash
  cd android && ./gradlew signingReport
  ```
  Copy the `SHA1` and `SHA-256` lines for the `debug` variant into Firebase Console тЖТ **Project Settings тЖТ Your apps тЖТ Android тЖТ Add fingerprint**.
- After adding fingerprints, **re-download `google-services.json`** and replace the file.

### 6. Android Gradle wiring
The Google Services plugin must be applied. Add the following if `flutter create` didn't:

**`android/build.gradle`** (project-level) тАФ inside `dependencies` of `buildscript`:
```gradle
classpath 'com.google.gms:google-services:4.4.1'
```

**`android/app/build.gradle`** тАФ at the very bottom:
```gradle
apply plugin: 'com.google.gms.google-services'
```

Also bump `minSdkVersion` to at least **23** in `android/app/build.gradle`:
```gradle
defaultConfig {
    minSdkVersion 23
    targetSdkVersion 34
    ...
}
```

### 7. Enable Firestore
- Firebase Console тЖТ **Firestore Database тЖТ Create database** тЖТ start in **production mode**
- Paste the rules from `firestore.rules` (included in this repo) and **Publish**.

### 8. Run the app
```bash
flutter run
```
On first launch you'll see the login screen тЖТ Google chooser тЖТ profile setup тЖТ home.

## ЁЯФР Firestore security rules

See `firestore.rules`. By default:
- A user can read/write **only their own** `/users/{uid}` document.
- Chats collection is locked until you build the chat feature (P1).

## ЁЯзк Troubleshooting

| Symptom | Fix |
| --- | --- |
| `ApiException: 10` on Google Sign-In | SHA-1/SHA-256 not registered in Firebase, or `google-services.json` is stale. Re-download. |
| `PlatformException(sign_in_failed)` | Make sure Google Sign-In is enabled in Firebase Auth providers. |
| `[firebase_core/no-app] No Firebase App` | `firebase_options.dart` still has placeholder values тАФ run `flutterfire configure`. |
| App stuck on splash | Firestore not enabled, or rules blocking the `users/{uid}` read. Check rules + console. |
